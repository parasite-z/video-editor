# video-editor (ฉบับภาษาไทย)

สกิล**ตัดต่อวิดีโอแบบสั่งงานด้วยภาษาธรรมชาติ** สำหรับ HeyGen [HyperFrames](https://hyperframes.heygen.com)

ในขณะที่ `/video-from-script` และ `/video-from-asset` คือการ "สร้างวิดีโอใหม่ตั้งแต่ต้นจนจบ" จากอินพุตเดียว — `/video-editor` จะให้ Claude ทำหน้าที่เป็น **นักตัดต่อที่นั่งอยู่หน้าไทม์ไลน์** กับวิดีโอที่มีอยู่แล้ว คุณสั่งงานด้วยคำพูดแบบไหนก็ได้ แล้วมันจะแปลงคำสั่งนั้นให้เป็นการแก้ไขจริงใน HyperFrames ลงมือแก้ ตรวจสอบ แล้วแสดงผลให้ดู

## ติดตั้ง (ครั้งเดียวต่อเครื่อง)

**ต้องมีก่อน:** [Claude Code](https://claude.com/claude-code), [Node.js 22 ขึ้นไป](https://nodejs.org) และ FFmpeg

จากนั้นรัน 2 บรรทัดนี้ใน terminal:

```bash
npx skills add heygen-com/hyperframes -g --all -y   # เอนจิน HyperFrames + สกิลที่เกี่ยวข้อง
npx skills add parasite-z/video-editor -g --all -y  # สกิลตัดต่อตัวนี้
```

เท่านี้ก็เรียก `/video-editor` ได้ในทุกเซสชันของ Claude Code — ถ้าจะอัปเดตภายหลัง ให้รันบรรทัดที่สองซ้ำ

> สำหรับเพื่อนร่วมทีมที่ไม่ถนัด terminal และอยากแค่ดับเบิลคลิก ดูโฟลเดอร์ [`team-installer/`](team-installer/) และคู่มือ [READ-ME-FIRST.th.md](team-installer/READ-ME-FIRST.th.md)

## วิธีเรียกใช้

เปิด Claude Code ในโฟลเดอร์โปรเจกต์ HyperFrames (หรือโฟลเดอร์ที่มีโปรเจกต์อยู่) แล้วพิมพ์:

```
/video-editor
```

จากนั้นสั่งงานเหมือนคุยกับนักตัดต่อได้เลย:

- "ตัด 2 วินาทีแรกออก"
- "ทำให้ฉาก 3 ช้าลง"
- "ใส่คำบรรยายตรงนี้ว่า 'สินค้าใหม่'"
- "เปลี่ยนเพลงประกอบเป็นไฟล์นี้"
- "ซูมเข้าไปที่สินค้า"
- "ทำให้เป็นแนวตั้งสำหรับ TikTok"
- "เพิ่มเสียงพากย์ให้ดังขึ้น"
- "ในวิดีโอนี้มีอะไรบ้าง?"
- "เรนเดอร์ออกมาเลย"

มันจะอยู่ในโหมดตัดต่อต่อเนื่อง — สั่ง แก้ แสดงผล วนไปเรื่อย ๆ จนกว่าคุณจะพอใจ

## ทำงานยังไง

มันเป็นเลเยอร์บาง ๆ ที่ทำหน้าที่ "นักตัดต่อ" วางทับสกิล HyperFrames ที่มีอยู่เดิม:

- **`hyperframes`** — โครงสร้างข้อมูล สไตล์ภาพ รูปแบบคำบรรยาย/เสียงพากย์/ทรานซิชัน
- **`hyperframes-cli`** — `init` / `lint` / `inspect` / `preview` / `render`
- **`hyperframes-media`** — `tts` (เสียงพากย์) / `transcribe` (ถอดเสียง) / `remove-background` (ลบพื้นหลัง)

สิ่งที่สกิลนี้เพิ่มเข้ามาคือ: การตีความคำสั่งแบบอิสระ, ตรรกะการจัดเรียงไทม์ไลน์ใหม่ (re-flow), และวงจร "ตรวจสอบก่อนบอกว่าเสร็จ" ส่วนกฎของเฟรมเวิร์กยังคงอยู่ในสกิลด้านบน — สกิลนี้แค่ตัดสินใจว่า *คำสั่งหนึ่ง ๆ หมายถึงการแก้อะไร* และ *การแก้นั้นส่งผลต่อไทม์ไลน์อย่างไร*

## ติดตั้งในโปรเจกต์อื่น

สกิลอยู่ใน `.claude/skills/` ดังนั้นจะใช้งานได้ในทุกเซสชันที่เปิดในโฟลเดอร์นี้ หากต้องการใช้ที่อื่น ให้คัดลอกหรือทำ symlink โฟลเดอร์ `.claude/skills/video-editor` ไปไว้ใน `.claude/skills/` ของโปรเจกต์นั้น หรือไว้ใน `~/.claude/skills/` เพื่อให้ใช้ได้ทั่วทั้งเครื่อง

> ต้องมีสกิลตระกูล `hyperframes` ติดตั้งไว้ (`npx hyperframes skills`), Node เวอร์ชัน 22 ขึ้นไป และ FFmpeg
